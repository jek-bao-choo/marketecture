WAS (Windows Process Activation Service)
Its main job is to manage the lifecycle of the worker processes (w3wp.exe).
Stopping WAS is a cascade shutdown. Because W3SVC depends on WAS, stopping the core was service forces Windows to automatically stop W3SVC and any other related services first. This single command reliably shuts down the entire IIS web engine. ^Du6xYzji

W3SVC (World Wide Web Publishing Service)
Its main job is to listen for incoming HTTP and HTTPS requests from clients (like web browsers).
Starting w3svc is a cascade startup. Because W3SVC needs was to run, telling Windows to start w3svc automatically makes it start the was service first. This ensures all dependencies are brought online in the correct sequence. ^JyP7elUe

The most specific setting for Datadog's DD_SERVICE, DD_VERSIOn, and DD_ENV would using the priority order. The priority order is: setting in the code first, then setting in the web.config file, and finally the system-wide environment variable. 

This means a service name set in your web.config will override any value set as an environment variable. Also because the web.config setting is specific to one application, setting in web.config, instead of env variables, is the best method for giving unique service names to multiple different apps running on the same server. But if cannot touch code or web.config then use Registry Editor and search for the Service then set it as Environment in the service. Datadog has a doc that shows how to set environment variable for a particular Windows Service. ^2HX1oUyC
