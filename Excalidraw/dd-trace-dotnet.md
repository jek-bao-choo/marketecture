WAS (Windows Process Activation Service)
Its main job is to manage the lifecycle of the worker processes (w3wp.exe).
Stopping WAS is a cascade shutdown. Because W3SVC depends on WAS, stopping the core was service forces Windows to automatically stop W3SVC and any other related services first. This single command reliably shuts down the entire IIS web engine. ^Du6xYzji

W3SVC (World Wide Web Publishing Service)
Its main job is to listen for incoming HTTP and HTTPS requests from clients (like web browsers).
Starting w3svc is a cascade startup. Because W3SVC needs was to run, telling Windows to start w3svc automatically makes it start the was service first. This ensures all dependencies are brought online in the correct sequence. ^JyP7elUe
